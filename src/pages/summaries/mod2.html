Frame Forwarding Concepts

  • Ingress: Frames entering the interface
  • Egress: Frames exiting the interface

Forwarding Behavior:

  • Based on ingress interface
  • Uses destination MAC address
  • References MAC address table
Important Note: Switches never forward traffic back out the receiving interface.

The Switch MAC Address Table

Forwarding Decision:

Uses destination MAC to select egress interface

Learning Process:

Dynamically learns MAC/port mappings

MAC Address Table (CAM):

  • Records source MAC addresses
  • Tracks ingress ports
  • Used for all forwarding decisions

The Switch Learn and Forward Method

Step 1: Learn – Examines Source MAC
  • Adds new MAC addresses to table
  • Resets 5-minute timeout for known MACs
Step 2: Forward – Examines Destination MAC
  • Forwards to known destination ports
  • Floods to all ports (except ingress) for unknown MACs

Switch Forwarding Methods

ASIC Processing: Hardware-based forwarding for maximum speed

Store-and-Forward Switching
  • Receives entire frame before forwarding
  • Performs CRC validation
  • Cisco's preferred method
Cut-Through Switching
  • Forwards after reading destination MAC only
  • Lower latency but no error checking

Collision Domains

Switch Benefit: Eliminates collision domains in full-duplex mode

Full Duplex Operation
  • No collisions possible
  • Simultaneous bidirectional communication
Half Duplex Operation
  • Collisions possible
  • Shared bandwidth contention
Default: Auto-negotiation enabled for duplex/speed on Cisco devices

Broadcast Domains

Definition: All Layer 2 devices receiving broadcast traffic

Key Characteristics
  • Only routers break broadcast domains
  • Switches flood broadcasts to all ports
Performance Impact
  • Excessive broadcasts cause congestion
  • Adds network latency

Switch Congestion Alleviation Features

Feature Description
Fast Port Speeds Up to 100Gbps (model dependent)
Fast Internal Switching High-speed internal bus architecture
Large Frame Buffers Temporary storage for traffic bursts
High Port Density
  • More local switching
  • Reduced backbone congestion
