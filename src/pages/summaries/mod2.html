Frame Forwarding Concepts

Ingress: Frames entering the interface

Egress: Frames exiting the interface

Forwarding Behavior:

  • The ingress interface
  • The destination MAC address
  • The switch's MAC address table
Important: A switch will never forward traffic back out the same interface it was received on.

The Switch MAC Address Table

Forwarding Decision: Based on destination MAC → selects egress interface.

Learning Process: Switch must learn MACs and port mappings.

MAC Address Table (CAM):

  • Records source MACs
  • Tracks ingress ports
  • Used for forwarding decisions

Learn and Forward Method

Step 1: Learn (Source MAC)
  • Adds MAC if new
  • Resets timeout if already exists
Step 2: Forward (Destination MAC)
  • Forwards to known MAC port
  • Floods frame if destination unknown

Switch Forwarding Methods

ASIC Processing: Uses custom hardware for ultra-fast frame forwarding.
Store-and-Forward Switching
  • Buffers full frame before forwarding
  • Performs CRC check (error detection)
  • Cisco preferred method
Cut-Through Switching
  • Forwards after reading destination MAC only
  • Lower latency, no error checking
  • Faster but can forward corrupted frames

Collision Domains

Benefit: Switches eliminate collisions → less network congestion.
Full Duplex
  • Two-way communication
  • No collisions
Half Duplex
  • Only one-way at a time
  • Collisions can occur
Default: Cisco devices auto-negotiate speed/duplex by default.

Broadcast Domains

Definition: A broadcast domain includes all devices receiving Layer 2 broadcasts.
Key Characteristics
  • Only routers break broadcast domains
  • Layer 2 switches flood broadcasts
Performance
  • Too many broadcasts = slower network
  • Increased congestion
Note: Sometimes called MAC broadcast domains.

Switch Features That Alleviate Congestion

Feature Description
Fast Port Speeds Up to 100Gbps per port depending on model
Fast Internal Switching High-speed internal bus or memory buffers
Large Frame Buffers Stores burst traffic temporarily
High Port Density
  • More local connections
  • Reduced congestion from inter-device traffic
