Cisco Switch Boot Sequence

  • Step 1: Runs POST (ROM) - Checks CPU, DRAM, and flash file system.
  • Step 2: Loads boot loader (ROM) - Runs after POST completes.
  • Step 3: Boot loader initializes CPU registers (memory mapping, size, speed).
  • Step 4: Boot loader sets up flash file system.
  • Step 5: Boot loader loads default IOS into memory and hands control to IOS.

The boot system Command

  • Auto-boot process: Uses the BOOT environment variable. If unset, loads the first executable file found.
  • Startup-config: IOS initializes interfaces using config.text (in flash).
  • Setting boot file: Use boot system flash:/path/to/file.bin and verify with show boot.

Switch LED Indicators

  • SYSTEM (SYST): Power and system status.
  • RPS: Redundant Power Supply status.
  • STAT: Port status (default).
  • DUPLX: Duplex status.
  • SPEED: Speed mode active.
  • PoE: Power over Ethernet status.
  • Mode Button: Toggles between indicator modes.

Recovering from System Crash

  1. Connect PC to console port.
  2. Unplug switch power cord.
  3. Reconnect and press Mode until SYST LED turns green.
  4. Boot loader prompt appears.

Boot loader allows:

  • Format flash
  • Reinstall IOS
  • Password recovery
  • File management with dir

Interface Error Types

  • Input Errors: Total inbound packet errors.
  • Runts: Too-small packets (<64 bytes).
  • Giants: Oversized packets (>1518 bytes).
  • CRC Errors: Mismatched checksums.
  • Output Errors: Transmission failure errors.
  • Collisions: Ethernet transmission overlaps.
  • Late Collisions: Occur after 512 bits transmitted.
