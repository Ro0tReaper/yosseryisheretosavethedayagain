VLAN Definitions

VLANs create logical networks with these key characteristics:

  • Segmentation: Groups devices on shared switches
  • Simplified Management: Organized device grouping
  • Traffic Isolation: Contains all frame types within VLAN
  • IP Addressing: Dedicated subnet per VLAN
  • Broadcast Control: Reduces domain size

Layer 2 Advantage: Provides Layer 3-style segmentation while operating at Layer 2.

VLAN Creation Example

Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Engineering
Switch(config-vlan)# exit

Benefits of VLAN Design

  • Smaller Broadcast Domains: Reduces broadcast traffic by dividing LANs
  • Improved Security: Limits communication to same-VLAN devices
  • IT Efficiency: Groups devices by function (e.g., faculty vs. students)
  • Cost Reduction: Single switch supports multiple VLAN groups
  • Performance Boost: Smaller domains increase available bandwidth
  • Simplified Management: Unified policies for similar groups

Design Tip: VLANs provide Layer 2 segmentation without requiring additional hardware.

Types of VLANs

Default VLAN (VLAN 1)

  • Preconfigured on all switches
  • Serves as:
    • Default Native VLAN
    • Default Management VLAN
  • Cannot be deleted or renamed

Security Recommendation: Reassign default features to other VLANs despite VLAN 1 restrictions.

Data VLAN

  • Carries user traffic (email/web)
  • VLAN 1 is default but should be changed

Native VLAN

  • Exclusive to trunk links
  • Untagged in 802.1Q trunks
  • Must match on both ends of trunk

Management VLAN

  • Dedicated to administrative traffic (SSH/Telnet)
  • Typically the SVI interface VLAN
  • Should be separate from user VLANs

Voice VLAN

  • Special requirements:
    • Guaranteed bandwidth
    • High QoS priority (DSCP EF)
    • <150ms end-to-end delay
  • Requires end-to-end network support

Defining VLAN Trunks

A trunk is a point-to-point link between two network devices with these Cisco-specific functions:

  • Multi-VLAN Support: Carries traffic for multiple VLANs simultaneously
  • Network-Wide Extension: Enables VLANs to span across the entire network infrastructure
  • Default VLAN Handling: Initially supports all VLANs (can be pruned)
  • Standards Compliance: Implements IEEE 802.1Q tagging protocol

Basic Trunk Configuration

Switch(config)# interface gigabitethernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk native vlan 99
Switch(config-if)# switchport trunk allowed vlan 10,20,30

VLAN Identification with 802.1Q Tags

  • Header Size: 4 Bytes total
  • FCS Recalculation: Required both when adding and removing tags
  • End Device Handling: Tags are stripped before delivery to hosts

802.1Q VLAN Tag Fields

Field Function
Type (TPID)
  • 2-Byte field (0x8100)
  • Tag Protocol Identifier
User Priority
  • 3-bit QoS priority value
  • Supports Canonical Format Indicator (CFI)
VLAN ID (VID)
  • 12-bit identifier (1-4094 usable)
  • Supports 4096 total VLANs

Technical Note: CFI bit (1-bit) enables Ethernet/Token Ring compatibility but is largely deprecated.

Native VLANs and 802.1Q Tagging

802.1Q Trunk Fundamentals

  • All VLANs are typically tagged except native VLAN
  • Native VLAN exists for legacy device compatibility
  • VLAN 1 is default native VLAN (should be changed)
  • Native VLAN must match on both ends of trunk
  • Different trunks can use different native VLANs

Security Warning: Never use VLAN 1 for production traffic due to security vulnerabilities.

Voice VLAN Tagging

  • VoIP Phone Operation: Acts as 3-port switch (uplink, PC, handset)
  • CDP Usage: Switch informs phone of Voice VLAN via CDP
  • QoS Handling: Tags voice traffic with Layer 2 CoS values
  • PC Traffic: May pass through untagged or transparently

Voice VLAN Tagging Behavior

Traffic Type Tagging Behavior
Voice VLAN
  • Always tagged
  • Includes Layer 2 CoS priority
Access VLAN (Tagged)
  • Optional tagging
  • Can carry CoS if tagged
Access VLAN (Untagged)
  • No VLAN tagging
  • No Layer 2 CoS

Configuration Example

Switch(config)# interface gig0/1
Switch(config-if)# switchport voice vlan 110
Switch(config-if)# switchport access vlan 10
Switch(config-if)# mls qos trust cos

* Enables CoS trust for VoIP traffic

Switch(config)# vlan 100

Change VLAN Port Membership

  • Change VLAN: Use switchport access vlan <vlan-id>
  • Reset to Default: Use no switchport access vlan to return to VLAN 1
  • Verify: Use show vlan brief or show interface fa0/18 switchport

Trunk Configuration Commands

  • Enter Global Config: configure terminal
  • Select Interface: interface <interface-id>
  • Set Trunk Mode: switchport mode trunk
  • Set Native VLAN: switchport trunk native vlan <vlan-id>
  • Allow Specific VLANs: switchport trunk allowed vlan <vlan-list>
  • Exit Config Mode: end

Dynamic Trunking Protocol (DTP)

  • DTP Modes: Use switchport mode with:
    • access: Forces access mode, negotiates access.
    • trunk: Forces trunk mode, negotiates trunking.
    • dynamic auto: Trunks if neighbor is trunk/desirable.
    • dynamic desirable: Actively tries to trunk.
  • Disable DTP: Use switchport nonegotiate to stop negotiation.
