Modern Network Security

Common Network Attacks

  • DDoS Attacks: Zombie networks overwhelm resources
  • Data Breaches: Theft of confidential information
  • Malware: Including ransomware like WannaCry

Security Devices

  • VPN Routers: Secure remote access
  • NGFWs:
    • Stateful inspection
    • NGIPS and AMP
    • URL filtering
  • NAC: Cisco ISE for AAA services

Endpoint Protection

  • Components:
    • AMP software
    • Email Security Appliance (ESA)
    • Web Security Appliance (WSA)
  • ESA Functions:
    • SMTP monitoring
    • Threat blocking
    • Email encryption
  • WSA Functions:
    • URL filtering/categorization
    • Malware scanning
    • Web traffic control

Security Integration: Cisco Talos provides real-time threat intelligence to ESA/WSA.

Network Authentication & AAA Security

Local Authentication Methods

  • Basic Password Authentication: Configured on console/vty/aux ports
  • SSH Authentication:
    • More secure remote access
    • Uses local username/password database
  • Limitations: Not scalable, no fallback mechanism

AAA Framework Components

  • Authentication: Verifies user identity
  • Authorization: Controls access privileges
  • Accounting: Tracks user activities

Authentication Methods

  • Local AAA:
    • Credentials stored on device
    • Ideal for small networks
  • Server-Based AAA:
    • Centralized credential management
    • Uses RADIUS/TACACS+ protocols
    • Scalable for large networks
    • Good when there are more than one switch/router

Authorization & Accounting

  • Authorization Features:
    • Automatic after authentication
    • Controls user privileges via attributes(ACLs)
  • Accounting Features:
    • Logs connection times and commands
    • Provides audit trails for security
    • Useful for troubleshooting

802.1X Port-Based Authentication

  • Components:
    • Supplicant (client device)
    • Authenticator (switch/AP)
    • Authentication server (RADIUS)
  • Function: Blocks unauthorized devices from accessing LAN ports

Security Insight: Server-based AAA with TACACS+ provides centralized control and detailed command logging for enterprise networks.

Layer 2 Security Vulnerabilities

The Layer 2 Weak Link

  • While Layers 3-7 are protected by firewalls/IPS/VPNs, Layer 2 is often exposed
  • Critical Risk: Compromising Layer 2 bypasses all upper-layer security
  • Modern threats amplified by BYOD policies and sophisticated attacks

Common Switch Attack Categories

  • MAC Table Attacks: Flooding attacks
  • VLAN Attacks: Hopping, double-tagging
  • DHCP Attacks: Starvation, spoofing
  • ARP Attacks: Spoofing, poisoning
  • Spoofing Attacks: MAC/IP address forgery
  • STP Attacks: Protocol manipulation

Layer 2 Mitigation Techniques

  • Port Security: Blocks MAC flooding & DHCP attacks
  • DHCP Snooping: Prevents starvation/spoofing
  • Dynamic ARP Inspection: Stops ARP attacks
  • IP Source Guard: Prevents address spoofing

Management Protocol Security

  • Secure Protocols: SSH, SCP, SFTP, SSL/TLS
  • Network Design:
    • Out-of-band management
    • Dedicated management VLAN
  • Access Control: Implement ACLs for management interfaces

Security Priority: Layer 2 vulnerabilities must be addressed first since they can bypass all upper-layer security measures.

Layer 2 Attack Methods & Mitigations

MAC Address Table Attack

How It's Done:

  • Floods switch with fake MAC addresses (e.g., using macof tool)
  • Overflows MAC table (8,000+ frames/sec)
  • Forces switch to flood frames to all ports

Mitigation:

  • Implement port security
  • Limit learned MAC addresses per port

VLAN Hopping Attacks

How It's Done:

  • Host spoofs 802.1Q/DTP signaling
  • Tricks switch into trunking mode
  • Gains access to all VLANs

Mitigation:

  • Disable trunking on access ports
  • Manually enable trunk links
  • Restrict native VLAN usage

DHCP Attacks

How It's Done:

  • Starvation: Tools like Gobbler lease all IPs
  • Spoofing: Rogue server gives false:
    • Gateways
    • DNS servers
    • IP addresses

Mitigation:

  • Implement DHCP snooping
  • Authorize legitimate DHCP servers

ARP Spoofing

How It's Done:

  • Sends gratuitous ARP Replies
  • Poisons ARP tables with fake MACs
  • Creates man-in-the-middle attacks

Mitigation:

  • Dynamic ARP Inspection (DAI)
  • DHCP snooping binding table

STP Manipulation

How It's Done:

  • Floods BPDUs with low bridge IDs
  • Becomes unauthorized root bridge
  • Intercepts all switched traffic

Mitigation:

  • BPDU Guard on access ports
  • Root Guard on designated ports

Pro Tip: Combine these mitigations with secure management protocols (SSH/SCP) and dedicated management VLANs for complete Layer 2 protection.
